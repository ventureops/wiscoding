# 프로젝트 운영 (Project Ops)

1인 또는 소규모 팀의 **프로젝트 운영 노하우**를 모읍니다.

코드를 짜는 것 외에 프로젝트를 **굴러가게** 만드는 모든 것 — 배포, 모니터링,
협업 규칙, 문서화, 의사결정 기록 등을 다룹니다.

## 다루는 주제

- **배포 루틴** — 정기 배포, 롤백 절차, 배포 전 체크리스트
- **CI/CD 셋업** — GitHub Actions, Vercel, 사전 훅, 자동 테스트 파이프라인
- **모니터링·알림** — 에러 트래킹, 성능 알림, 비용 알림
- **이슈·PR 운영** — 라벨 체계, 템플릿, 리뷰 규칙, 머지 정책
- **문서화 정책** — README 구조, ADR(아키텍처 결정 기록), 변경 로그
- **협업 규칙** — 브랜치 전략, 커밋 메시지 컨벤션, 코드 리뷰 가이드
- **온콜·당직** — 작은 팀에서의 장애 대응 루틴
- **비용·리소스 관리** — 클라우드 비용 추적, 무료 티어 활용
- **법·라이선스** — OSS 라이선스, 개인정보 처리, 저작권 표기

## 자료 추가하기

이슈 템플릿 **"새로운 팁"** 에서 분류를 `프로젝트 운영`으로 골라 제출하시거나,
직접 PR로 이 폴더에 마크다운 파일을 추가해주세요.

파일명 예시:
- `deploy-checklist-vercel.md`
- `pr-template-and-labels.md`
- `adr-lightweight-template.md`

자세한 작성 규칙은 루트의 [CONTRIBUTING.md](../../CONTRIBUTING.md) 참고.
